重要的时候要说三遍！！！

首先，下载需要的软件

  • 安装Node.js:用于生成动态界面。点我下载
  • 安装git:下载一系列插件以及后期的上传下载。点我下载

设置github

  • 已经有账号的，就新建一个Repository，注意Repository名字要和用户名一致。
  • 打开Repository，复制ssh的链接，例：git@github.com:guojjing/guojjing.github.io.git

使用hexo部署博客

  • 在除了c盘的随便一个盘的位置创建一个名为blog的文件夹。
  • 在blog里右击Git Bash Here
  • 输入npm install -g hexo ，下载hexo,网速不好会出现ERR错误，重试就好。成功示例:

    下同。

  • 输入hexo init 下载hexo的初始化文件

  • 输入npm install 安装本地博客所需插件

  • 输入npm install hexo-deployer-git –-save 安装hexo部署程序

  • 输入hexo generate 缩写 hexo g 生成静态页面至public目录

  • 输入hexo server 开启本地预览访问端口(localhost:4000 “ctrl + c” 关闭server)

  • 打开blog文件夹中的_config.yaml文件，下滑到最后，
    修改配置为：
    type: git
    repo: git@github.com:xxxx/xxxx.github.io.git
    branch: master

设置ssh密钥

  • 如果已经有了密钥，则直接删除.ssh文件夹中的所有文件，重新获取密钥（如果绑有重要网站的，慎重决定）

  • 如果不想删除，验证一下ssh，如果有效，就使用当前ssh key,输入ssh -T git@github.com ，回复yes
    成功示例:

  • 重获密钥，ssh-keygen -t rsa -C “邮箱” 一路回车

  • 输入ssh-agent -s

  • 输入ssh-add ~/.ssh/id_rsa如果出现如下错误示例:

    则继续输入 eval `ssh-agent -s` ，再输入ssh-add
    成功示例:

  • 输入cat ~/.ssh/id_rsa.pub 导出密钥

  • 在github中，点击右上角头像，点击setting

  • 点击ssh key中的add keys，添加。

  • 输入ssh -T git@github.com ，回复yes验证密钥

完成部署

  • 输入hexo g

  • 输入hexo d

  • 最后在浏览器输入 用户名.github.io 成功示例:

上传博客

  • 在blog文件的_config.yaml文件中配置博客的标题等信息示例：

  • 然后hexo常用的几个命令

  1. hexo new “文件名” 在blog\source_posts中，创建新的md文件

  2. hexo clean 清理文件，为上传博客做基础

  3. hexo g 在public中生成目标的html

  4. hexo d 发布博客

输入自己的网址看效果

在部署完博客后，可能没有立即出现预期效果，等一会刷新或者换别的浏览器、电脑就好。
